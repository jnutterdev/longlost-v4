It's been awhile. And the directions I've taken in the past couple of months are... kind of all of the map. That is, each week/weekend, I start digging into a new language, or overhaul a laptop with another Linux distro. This is all well and good, but not productive for the long run. 

That being said, I'm starting to put together a plan for the future. At least, one step at a time. Starting small for now so as not to get overwhelmed, but I plan on adding to this list as I go. So here it goes:

- Get Linux certification
- Start and finish one project using an interpreted language

Right now, I'm thinking that the project will be Ruby based. The more I read up on the language, and tinker with it, the more I like it. Python is great, but there's just something about it that doesn't speak to me. I know, I know, give it more time. 

PHP is good, too. And I like working with it; it just makes a lot of sense to me. I don't really have an excuse for not continuing my projects with it, other than just having to spin up a server to test out the code each time. I'll circle back to it in due time, but I want to give Ruby a shot for awhile, which will be good to round everything out.
